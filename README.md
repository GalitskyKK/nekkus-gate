# Nekkus Gate

DNS-уровень блокировка: реклама и трекеры. Локальный DNS-сервер отдаёт NXDOMAIN для доменов из блок-листа, остальные запросы форвардит на upstream (8.8.8.8).

## Сборка

```bash
go build -o nekkus-gate ./cmd
```

Кроссплатформа: Windows, Linux, macOS. Зависимости — только стандартная библиотека и `miekg/dns`, без cgo для основного бинарника (cgo может требоваться только при сборке с GUI/desktop из nekkus-core).

## Запуск

- **Standalone:** `./nekkus-gate` — поднимает HTTP (9003), gRPC (19003), DNS (UDP 5354 по умолчанию; 5353 часто занят mDNS в Windows).
- **Из Hub:** Hub передаёт `--mode=hub`; Gate работает без окна, только API и DNS.

## Блок-лист

Файл `blocklist.txt` в директории данных (по умолчанию `~/.config/nekkus/gate` на Linux, `%APPDATA%\nekkus\gate` на Windows). Один домен на строку, строки с `#` — комментарии. Пример:

```
doubleclick.net
google-analytics.com
facebook.com
```

Поддомены блокируются автоматически (например, `a.b.tracker.com` при наличии `tracker.com` в списке).

## DNS-фильтр (включить/выключить)

Запускайте Gate **от имени администратора** (Windows) или **root** (Linux/macOS). В интерфейсе есть кнопка **«Включить»** / **«Выключить»**:

- **Включить** — сохраняются текущие DNS (DHCP или статические), в систему выставляется `127.0.0.1`, DNS-сервер Gate переключается на порт 53. Весь трафик резолвится через Gate.
- **Выключить** — системный DNS восстанавливается (автоматически/DHCP или сохранённые серверы), Gate снова слушает порт 5354.

После сбоя или закрытия Gate при следующем запуске настройки DNS восстанавливаются автоматически, если остался файл бэкапа.

Если порт 53 занят (например, службой «Кэширование DNS»), Gate автоматически включает блокировку через **файл hosts**: домены из блок-листа дописываются в `hosts` (0.0.0.0), системный DNS не меняется, ничего останавливать не нужно. В интерфейсе будет показано «Фильтр включён (через файл hosts)». При «Выключить» файл hosts восстанавливается из бэкапа.

## API

- `GET /api/stats` — для виджета Hub: `total_queries`, `blocked_today`, `blocked_total`, `blocked_percent`, `blocklist_count`.
- `GET /api/health` — проверка живости.
- `GET /api/filter/status` — `active` (фильтр включён?), `port` (53 или 5354).
- `POST /api/filter/enable` — включить DNS-фильтр (нужны права админа).
- `POST /api/filter/disable` — выключить и восстановить системный DNS.
